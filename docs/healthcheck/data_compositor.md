我的程序设计思路如下：
1.我会接收来自controller.c的Data数组和用户名username。Data中含有我们之前使用libcrul爬取的json
2.首先我先打印contestlist（这一步已经优化到直接打印爬下来的原文了），并建立一个contestlist的cJSON对象以在后续使用
3.然后我会统计之前爬取的user.rating中含有的比赛的数量，遍历他的cJSON，并建立一个RatingChange数组对应这里面的数据
4.然后统计submissions，遍历他的cJSON，建立Submission数组对应数据
5.接着我创建ContestRecord数组，把用户参加的比赛的记录和对应比赛（contestId相同即为对应比赛）和他的submission放在一起
6.通过时间关系判断是否迟交
7.以JSON形式输出迟交的记录
